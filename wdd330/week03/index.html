Creating Regular Expressions:

There are two ways to create a regular expression.The first, and preferred way, is to use the literal notation of writing the regular expression between forward slashes that we’ve already seen: 

      const pattern = /[a-zA-Z]+ing$/;

Alternatively, you can create a new instance of the RegExp object using the new operator and a constructor function: 

      const pattern = new RegExp('[a-zA-Z]+ing');

Events

Events is another part of DOM (Document Object Model). It provides a link between the web page and the user interactions (clicking a link, pressing a key, or moving a mouse) which can be detected by the program and respond to. JavaScript uses event listeners to listen out for any clicks on the page. Everytime page is clicked, a callback function is called. This allows the program so do other things while it waits for the click event (click from the mouse, press of a key, or taps on the screen). 

    document.body.addEventListener("click", doSomething);

Inline Event handler: Inline attributes is the original way events are dealt with in the browser but is not recommended found on page 261. 

    <p onclick="console.log('You Clicked Me!')">Click Me</p>

The event listeners is the current standard to dealing with events. method is called on a node object, the node to which the event listener is being applied. 

    document.body.addEventListener('click',doSomething);

The addEventListener() method can also be called without a node, in which case it is applied to the global object, usually the whole browser window. In this example. we are adding a click event listener to the whole page. This is because addEventListener() method is called witout a node referenced after it. 

    addEventListener('click', () => alert('You Clicked!'));

Another way of doing the same thing is by creating/declaring a function and reference it in the event listenr. 

    function doSomething() {
      alert('You Clicked!');
    }
    
    addEventListener('click',doSomething);
